## gateway

- Добавлен hotelCacheDataSource.js
	Кастомный `RemoteGraphQLDataSource` , реализующий кэш и батчинг для hotel-subgraph
	Необходим, чтобы уменьшить количество вызовов резолверов Hotel в случае, если один пользователь совершал бронирования в одном отеле несколько раз

## promo

- Добавлен сабграф и обвязка для него (Dockerfile, package.json, index.js)
  Типы схожи с тем, что указаны в задании, но немного отличаются
	```
	extend type Booking @key(fields: "id") {
		id: ID! @external
		promoCode: String @external
		discountPercent: Int @external
		discountInfo(userId: ID): DiscountInfo! @requires(fields: "promoCode")
	}
	type DiscountInfo {
		isValid: Boolean!
		originalDiscount: Float! # Исходное значение из booking
		finalDiscount: Float! # Актуальное значение после проверки
		description: String
		expiresAt: String
	}
	type Query {
		validatePromoCode(code: String!, userId: ID): DiscountInfo!
	}
	```

	`Booking` расширяется полем `discountInfo`, строящемся на основе значения `promoCode`

См скрины в `3. Успешный вызов`

====
	- `Booking` - расширяет Booking данными `discountInfo`. Проверяется промокод, его валидность для пользователя, у которого проверяются бронирования и в случае валидного кода он применяется
	- `validatePromoCode` - проверяет валидность промокода для конкретного пользователя

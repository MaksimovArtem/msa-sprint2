В задаче использовался mock booking-service из task4
	- В README сказано, что необязательно тащить свой
	- Время сборки мокового модуля значительно меньше, чем полноценного booking-service
	- booking-service реально не нужна коллекция /ping, а быстро придумать что-то стоящее и наглядное в качестве экспериментальной фичи не вышло

## booking-service

- В Dockerfile добавлено пробрасывание порта 8080
- check-dns из корня скопирован в папку booking-service для сборки контейнера
- main.go расширен
	- Наличие флага проверяется и логируется
	- В случае `ENABLE_FEATURE_X="true"` вызов `/ping` возвращает `pong-feature`
	- В случае `ENABLE_FEATURE_X="false"` вызов `/ping` возвращает `pong` 

## helm

	- `replicaCount = 1`
	- `pullPolicy = Never`
	- Выделены (совсем) небольшие ресурсы и лимиты
	- Добавлены параметры для `livenessProbe` и `readinessProbe`
	- Добавлена конфигурация для значения `ENABLE_FEATURE_X`
- Добавлен `values-prod.yml` (ниже дельта относительно `values-staging.yml`)
	- `replicaCount` увеличен до 2
	- `pullPolicy = IfNotPresent` (для оптимизации на проде)
	- Ресурсы и лимиты увеличены
	- Экспериментальная фича - отключена
- Расширен `templates/deployment.yaml` для контейнера `booking-service`

	- Добавлена секция `resources`
